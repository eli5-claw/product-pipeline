## Rivet
• **Summary:** Serverless primitive for stateful backends using Actors — long-lived processes that keep state in memory alongside compute.
• **Link:** https://github.com/rivet-dev/rivet
• **Category:** Distributed Systems / Developer Tools
• **Why it matters:** Rivet eliminates database round-trips by keeping state in memory with compute. Each actor has built-in state, storage, workflows, scheduling, and WebSockets. It's like Cloudflare Durable Objects but open-source and multi-platform (works with Vercel, Railway, Kubernetes, AWS ECS). The "hibernation" feature allows actors to sleep when idle and wake instantly with no cold starts.

## PageIndex
• **Summary:** Vectorless, reasoning-based RAG system that builds hierarchical tree indexes from documents for human-like retrieval.
• **Link:** https://github.com/VectifyAI/PageIndex
• **Category:** AI/ML Infrastructure
• **Why it matters:** PageIndex challenges the vector database paradigm by using document structure and LLM reasoning instead of semantic similarity. It achieved state-of-the-art 98.7% accuracy on FinanceBench. The approach simulates how human experts navigate complex documents through tree search — offering better explainability and traceability than opaque "vibe retrieval" vector search.

## Scrapling
• **Summary:** Adaptive web scraping framework with AI-powered element tracking that survives website design changes.
• **Link:** https://github.com/D4Vinci/Scrapling
• **Category:** Developer Tools / Web Scraping
• **Why it matters:** Scrapling's parser learns from website changes and automatically relocates elements when pages update. It bypasses anti-bot systems like Cloudflare Turnstile out of the box and includes a built-in MCP server for AI-assisted scraping. The "adaptive" feature means scrapers don't break when sites redesign — a persistent pain point in web scraping.

## GitNexus
• **Summary:** Zero-server code intelligence engine that indexes codebases into knowledge graphs for AI agent context.
• **Link:** https://github.com/abhigyanpatwari/GitNexus
• **Category:** Developer Tools / AI Infrastructure
• **Why it matters:** GitNexus builds a knowledge graph of every dependency, call chain, and execution flow in your codebase, then exposes it through MCP tools to AI agents. It prevents agents from "missing dependencies, breaking call chains, and shipping blind edits." The client-side approach (everything runs locally or in-browser) ensures privacy while giving even smaller models full architectural clarity.
